Important correction:
Do NOT create Qdrant memory immediately when burstRole === 'final'.

Why:
The current 'final' is tentative.
A later task_rescheduled event within the 5-minute burst window can retroactively turn the previous final into member.
If Qdrant memory is written immediately, it may become stale or false.

Required Phase 4 principle:
Only a settled reschedule burst may create one factual Qdrant memory.

Definition:
A task_rescheduled event is "settled" only if:
- metadata.burstRole is 'final' or 'sole'
- isInitialPlacementRefinement is false
- occurredAt is older than BURST_WINDOW_MS
- there is no later task_rescheduled event for the same userId + taskId + dateIso within the burst window
- it has not already produced a settled reschedule memory

Product rules:
1. Raw task_rescheduled events remain preserved.
2. Intermediate burst members do not write Qdrant memory.
3. Initial placement refinement does not write normal reschedule memory.
4. Settled burst final may write exactly one factual positional memory.
5. No behavioral conclusion:
   - no postponement
   - no avoidance
   - no planning failure
   - no procrastination
   - no pattern
6. Repeated burst patterns across days belong to a future Pattern Learning phase, not Phase 4.
