Sprint Pack 3ה בפרויקט Synco:
Trust + QA Pack מלא לתרחישי אמון קריטיים.

מטרה:
לבנות ולהריץ בדיקות אמון מקיפות על כל ליבת סינקו שנבנתה עד עכשיו, בלי להוסיף פיצ׳רים גדולים.
המטרה היא לוודא שסינקו מתנהג בצורה אמינה, לא ממציא זיכרון, לא משבץ בעבר, לא משנה DB בלי אישור, ומציג למשתמש את הזמנים בצורה ברורה.

חלק 1: יצירת Trust QA script

צור קובץ בדיקות חדש:

scripts/synco-trust-qa.sh

או אם הפרויקט משתמש ב-node scripts:

scripts/synco-trust-qa.ts

הסקריפט צריך להריץ סדרת בדיקות HTTP מול השרת המקומי.

הוא צריך לבדוק:

1. Health
GET /api/health

2. No Past Scheduling
שלח POST /api/planner/schedule עם:
- date = היום
- nowIso = היום בשעה 17:20
- dayStart = 08:00
- dayEnd = 22:00
- 3 משימות קצרות ללא שעה

ציפייה:
- אף scheduledTask לא מתחילה לפני now + 5 דקות
- response כולל nowContextSummary או warning מתאים
- אין שיבוץ בעבר

3. Anchor Context
שלח tasks:
- יציאה ליהוד ב-18:30, fixed
- לארגן תיק ליציאה
- להחליט עם שני האם קיטו מגיע
- להעיר את שני לפני יציאה

ציפייה:
- prep tasks לפני anchor
- prep tasks לא בבוקר אם now אחר הצהריים
- schedulingContextReasons קיימים
- anchorContextSummary קיים

4. False Learning Prevention
צור משימה חדשה ללא events.
שלח ל-planner/schedule.

ציפייה:
- learningBoost = 0
- learningReasons = []
- אין טקסט שאומר שהמשימה זזה בעבר

5. Learning source filtering
צור LearningEvents עם source:
- test
- seed
- debug
- manual_test

ציפייה:
- לא משפיעים על learningBoost
- לא משפיעים על durationSuggestion

6. Real Learning Boost
צור 3 אירועי task_rescheduled אמיתיים לאותו taskId.
שלח את אותה משימה ל-planner/schedule.

ציפייה:
- learningBoost מתקבל רק לאותו taskId
- task עם title דומה אבל id אחר לא מקבל boost

7. Duration Suggestion safety
צור 3 אירועי task_execution_completed אמיתיים עם actualDurationMinutes.
שלח task דומה.

ציפייה:
- durationSuggestion מופיע
- durationMinutes המקורי לא משתנה
- schedule times לא משתנים אוטומטית בגלל ההצעה

8. Weak title no suggestion
שלח task עם title דומה חלש.

ציפייה:
- אין durationSuggestion

9. Draft no DB before commit
שלח Draft Preview דרך planner/schedule.

ציפייה:
- לא נוצרת UserTask ב-DB
- לא נוצרת שמירה לפני commit

10. Commit draft schedule
שלח commit-draft-schedule עם draftTasks תקינים ו-scheduledItems תקינים.

ציפייה:
- נוצרות UserTasks אמיתיות
- duration המעודכן נשמר
- task_created events נוצרים
- schedule_applied נוצר

11. Commit rollback
שלח commit-draft-schedule עם draftId חסר או scheduledItem לא תואם.

ציפייה:
- rollback מלא
- לא נוצרת חצי משימה
- response ברור

12. Apply schedule existing tasks
בדוק planner/apply-schedule למשימות קיימות.

ציפייה:
- שמירה רק אחרי אישור
- completed/deleted לא מתעדכנות

13. Execution start persistence
בדיקה ככל שניתן ברמת unit או store:
- startTaskExecution
- executionStartTimes נשמר
- completeTask מוחק אותו
- duplicate start לא יוצר שני events

14. Execution expiry
הכנס executionStartTime ישן מעל 24 שעות.

ציפייה:
- לא מחושב duration ענק
- metadata actualStartSource = expired_execution_start_fallback
- entry נמחק

15. Local time display contract
בדוק שה-client שולח:
- nowIso
- timezoneOffsetMinutes
- userTimeZone

וב-response יש מספיק מידע כדי להציג שעות מקומיות.
אם יש פער UTC מול Local, דווח.
