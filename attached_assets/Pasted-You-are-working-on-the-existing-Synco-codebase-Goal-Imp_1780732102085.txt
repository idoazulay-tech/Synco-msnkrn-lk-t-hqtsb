IMPORTANT PRODUCT RULE

Do NOT write a Qdrant memory immediately when burstRole === 'final'.

Why:
burstRole='final' is tentative.
A later task_rescheduled event within the 5-minute burst window can retroactively turn the previous final into member.
A Qdrant memory may be written only after the reschedule burst has settled.

DEFINITION: SETTLED RESCHEDULE

A task_rescheduled event is eligible for settled memory only if:

1. metadata.burstRole is 'final' or 'sole'
2. metadata.isInitialPlacementRefinement === false
3. event.occurredAt is older than BURST_WINDOW_MS
4. There is no later task_rescheduled event for the same:
   - userId
   - taskId
   - dateIso
   within BURST_WINDOW_MS after this event
5. metadata.settledMemoryWrittenAt is not present
6. event has enough factual data to build a safe memory text

MEMORY RULES

Allowed:

Implement:

1. findSettledRescheduleEvents(userId: string, opts?)

Find task_rescheduled rows where:
- userId matches
- eventType = 'task_rescheduled'
- metadata.burstRole is 'final' or 'sole'
- metadata.isInitialPlacementRefinement is false
- metadata.settledMemoryWrittenAt is missing
- occurredAt < now - BURST_WINDOW_MS
- if opts.taskId exists, filter taskId
- if opts.dateIso exists, filter dateIso

Then for every candidate, verify there is no later task_rescheduled event with:
- same userId
- same taskId
- same dateIso, if dateIso exists
- occurredAt > candidate.occurredAt
- occurredAt <= candidate.occurredAt + BURST_WINDOW_MS

Only return candidates that pass this "no later movement" check.

If metadata is awkward to filter directly in Prisma, it is acceptable to fetch a bounded set of task_rescheduled candidates older than BURST_WINDOW_MS and filter metadata in TypeScript.
Keep it safe and clear.

2. buildSettledRescheduleMemory(event)

Return null if:
- burstRole is not 'final' or 'sole'
- isInitialPlacementRefinement is true
- taskTitleSnapshot is missing
- event is not task_rescheduled

Build factual Hebrew text:

If fromStartTime and toStartTime exist:
"המשתמש שינה את זמן המשימה {title} מ־{HH:MM} ל־{HH:MM} בתאריך {dateIso}."

If fromStartTime or toStartTime is missing:
"המשתמש שינה את זמן המשימה {title} בתאריך {dateIso}."

Important:
Do not use forbidden words.
Do not imply why the task moved.
Do not write any conclusion.

Return a derived memory object with:
- sourceLearningEventId: event.id
- memoryKind: 'reschedule_settled'
- schemaVersion: 1
- content
- deterministic pointId based on:
  event.id + ':' + memoryKind + ':v' + schemaVersion
- metadata:
  source: 'learning_event'
  memoryKind: 'reschedule_settled'
  schemaVersion: 1
  integrityStatus: 'accepted_fact'
  status: 'active'
  eventType: 'task_rescheduled'
  burstRole
  isInitialPlacementRefinement: false
  evidenceType: 'settled_reschedule_burst'
  sourceLearningEventId: event.id
  taskId
  dateIso
  fromStartTime
  toStartTime
  fromEndTime
  toEndTime
  importance: 'low'

3. processSettledReschedules(userId: string, opts?)

Flow:
- if process.env.SYNCO_SETTLED_DERIVATION_ENABLED === 'false':
  return safely with disabled status
- find settled candidates
- for each candidate:
  - build memory
  - if null:
    - if isInitialPlacementRefinement true:
      update metadata with:
        placementRefinementSkippedAt
        settledMemorySkipReason: 'initial_placement_refinement'
    - continue
  - call storeUserMessage with:
    userId
    memory content
    metadata including pointId
  - await this Qdrant write
  - only after successful write:
    update the candidate LearningEvent metadata:
      settledMemoryWrittenAt: new Date().toISOString()
      settledMemoryKind: 'reschedule_settled'
      settledMemoryPointId: pointId
- if Qdrant write fails:
  - do not update settledMemoryWrittenAt
  - count error
  - continue with next candidate
- return summary:
  { processed, skipped, errors, candidates }
