PRODUCT RULE

Synco should ask now only when missing information blocks the requested action.

If missing information is useful for learning but not critical to the current action:
- Execute the safe action now.
- Create a deferred open question.
- Let the user answer later in "מרכז ההיכרות" / "סינקו שואל".

Examples:
User says:
"תזכיר לי לדבר עם דניאל על הפרויקט בשבת"

Synco should:
- Create/schedule the task.
- Create deferred questions like:
  - "מי זה דניאל עבורך?"
  - "על איזה פרויקט מדובר?"
- Not block task creation.

PART 2 — SERVICE LAYER

Create:

server/brain/services/openQuestions.ts

Implement functions:

createOpenQuestion(input)
listOpenQuestions(userId, opts?)
answerOpenQuestion(questionId, userId, answerText, opts?)
dismissOpenQuestion(questionId, userId)
snoozeOpenQuestion(questionId, userId, until?)

Rules:

createOpenQuestion:
- Create only if questionText is non-empty.
- Default status = open.
- Default blocking = false.
- Default priority = normal.
- Truncate sourceInputText to max 100 chars.
- Do not create duplicate identical open questions for same:
  userId + questionType + relatedEntityName + relatedTaskId + questionText
- If duplicate exists, return the existing question instead of creating a new one.

listOpenQuestions:
- Return status='open' by default.
- Sort by priority and createdAt.
- Do not return expired questions unless explicitly requested.

answerOpenQuestion:
- Validate non-empty answerText.
- Set:
  status='answered'
  answeredAt=now
  answerText
  answerSource='user_explicit' by default
  answerConfidence=1.0 by default
- Do not write to Qdrant yet in Phase 1.
- Do not generate follow-up questions yet in Phase 1.

dismissOpenQuestion:
- status='dismissed'

snoozeOpenQuestion:
- status remains open
- set expiresAt or snoozedUntil if there is already a preferred existing pattern.
- If no snooze field exists, use expiresAt cautiously or report that a dedicated snoozedUntil field is needed.
- Do not overcomplicate.
