אתה מפתח ב Replit את מערכת MA. אני רוצה שתבנה “Temporal Engine” מלא לשפה עברית (וגם תמיכה חלקית באנגלית שמופיעה בתוך עברית), שמבין זמן כמו בן אדם ומחזיר ייצוגים טכניים יציבים לשיבוץ בלוז.

היעד:
המשתמש כותב או מדבר בעברית חופשית (כולל זמנים מדויקים, תאריכים, משכים, טווחים, חזרתיות וזמנים עמומים).
המערכת מפענחת, מנרמלת, מחשבת חלונות פנויים בלוז, ומשבצת או מציעה שיבוץ בצורה אינטואיטיבית.
אסור לנחש כשאין בסיס. כשאין מספיק נתונים, יש להחזיר needs_clarification עם שאלת הבהרה אחת ממוקדת.

שפת היעד: עברית.
אנגלית: לתמוך רק בביטויים שנכנסים לשיחה בעברית (לדוגמה: deadline, standup, ASAP, AM/PM, 3pm, 14:00) בלי להפוך את המנוע לאנגלי.

אזור זמן ברירת מחדל: Asia/Jerusalem.
המערכת מקבלת now (תאריך ושעה נוכחיים) ומחשבת יחסית אליו.

חובה לבנות מודול חדש:
ma_temporal_engine_he_v1

תוצרים חובה:
1) מודול קוד נקי ומודולרי
2) סט בדיקות יחידה גדול ומקיף
3) דוגמת שימוש (CLI או endpoint) שמדגים קלטים בעברית ופלט JSON
4) תיעוד קצר במבנה README: מה המודול עושה, איך קוראים לו, דוגמאות קלט פלט

פלט אחיד (JSON) שהמודול מחזיר:
- TimePoint:
  { type:"timepoint", date?:"YYYY-MM-DD", time:"HH:MM", timezone:"Asia/Jerusalem", confidence:0..1, sourceText, reason?, needs_clarification?:boolean, question?:string }
- DatePoint:
  { type:"datepoint", date:"YYYY-MM-DD", confidence, sourceText, reason?, needs_clarification?, question? }
- Duration:
  { type:"duration", duration_minutes:number, duration_iso?:"PT..", confidence, sourceText, reason?, needs_clarification?, question? }
- Interval:
  { type:"interval", start?:"YYYY-MM-DDTHH:MM", end?:"YYYY-MM-DDTHH:MM", window?:"broad|medium|tight", confidence, sourceText, reason?, needs_clarification?, question? }
- Recurrence:
  { type:"recurrence", pattern:{freq:"daily|weekly|monthly", byDay?:string[], interval?:number, count?:number, until?:string}, confidence, sourceText, reason?, needs_clarification?, question? }

בנוסף, המודול חייב להחזיר גם “TemporalHints” עבור זמנים עמומים:
{ hints:{ preferredWindows?: Array<{label, startHour, endHour}>, softness?: "hard|soft", latest?: "YYYY-MM-DDTHH:MM" }, ... }

חובה לתמוך בהבנה מלאה של:
A) שעות מדויקות בעברית מדוברת:
- "שמונה חמישים ותשע" => 8:59
- "שמונה חמישים" => 8:50
- שעה עגולה: "שמונה אפס אפס", "שמונה אפס", "שמונה בדיוק", "שמונה עגול", "שמונה על הדקה" => 8:00
- "שתיים ושלוש" (כשמובן שזה שעה ואז דקות) => 2:03
כל הדקות 00 עד 59, תמיד להחזיר דקות בשתי ספרות.

B) רבע, חצי, חסר:
- "עשר ורבע" => 10:15
- "עשר וחצי" => 10:30
- "רבע לשמונה" => 7:45
- "שלוש חסר רבע" => 2:45
וגם "חמש לאחת" => 12:55, "עשר לאחת" => 12:50 אם מופיע.

C) חלקי יום והקשר AM/PM:
- בבוקר, בצהריים, אחר הצהריים, בערב, בלילה, לפנות בוקר
דוגמה: "8 בערב" => 20:00, "8 בבוקר" => 08:00
אם אין הקשר והזמן 1-12, אל תכפה 24 שעות, הורד confidence או שאל הבהרה רק אם חייב.

D) תאריכים יחסיים:
- היום, מחר, מחרתיים, אתמול, שלשום
- "בעוד X", "תוך X", "עוד X"
להחזיר DatePoint או Duration בהתאם להקשר.

E) ימי שבוע:
- יום ראשון עד שבת
- קיצורים: יום א', א', ב' וכו
- "שני הקרוב", "שני הבא", "שני הזה"
כללים: לבחור את המופע הבא קדימה בזמן. אם נאמר "הקרוב" ביום עצמו, להתייחס להיום אלא אם ההקשר אומר אחרת.

F) משכי זמן:
- דקה, שתי דקות, 10 דקות
- רבע שעה, חצי שעה
- שעה, שעתיים, שלוש שעות
- יום, יומיים, שבוע, שבועיים, חודש, חודשים
נרמול ל duration_minutes (וגם duration_iso אם נוח)

G) טווחים:
- "מ 8 עד 10"
- "בין 16 ל 18"
- "מחר מ 8 עד 10"
להחזיר Interval עם start/end אם אפשר. אם חסר תאריך, תן Interval של חלון זמן בלבד עם window label וביטחון מתאים.

H) חזרתיות:
- כל יום, כל שבוע, כל יום שני, פעמיים בשבוע, כל יומיים, כל חודש ב 5
להחזיר Recurrence pattern. אם חסרים פרטים, needs_clarification.

I) זמנים עמומים:
המנוע חייב לזהות ולהחזיר “חלון מועדף” במקום שעה מדויקת כשזה מה שהמשתמש אמר:
- בערך, לקראת, בסביבות, מתישהו, כשיהיה זמן, מאוחר יותר, בהמשך, סוף היום, סוף השבוע, תחילת השבוע
התנהגות חובה:
1) לא להמציא שעה ספציפית בפלט של המנוע כשזה עמום.
2) להחזיר hints של חלון זמן, רמת קשיחות (softness) וגבול אחרון אם משתמע.
3) להשאיר לשכבת השיבוץ (scheduler) לבחור חריץ פנוי בהתאם ללוז.

חיבור לשכבת שיבוץ:
תייצר פונקציה נוספת במודול:
suggestScheduleSlots(temporalResult, calendarFreeBusy, taskDurationMinutes, rules) -> suggestions[]
שמטרתה:
- לקבל חלונות פנויים (free/busy)
- להציע 1-3 אפשרויות שיבוץ
- לכבד חוקים: שעות שינה, לא לשבץ בלילה אלא אם הוגדר, מינימום מרווח, לא להעמיס

הקוד חייב להיות מודולרי (פונקציות חובה):
- tokenize(text): פיצול טקסט, ניקוי סימנים, טיפול בגרשיים וקיצורים
- normalizeTokens(tokens): איחוד ו וריאציות ("ושלוש" => "ו","שלוש")
- parseNumericTime(tokens): שעה ודקות מדוברת כולל אפס אפס
- parseQuarterHalf(tokens): ורבע, וחצי, רבע ל, חסר רבע
- parseDayParts(tokens): בוקר ערב וכו
- parseRelativeDates(tokens, now): היום מחר וכו
- parseWeekdays(tokens, now): יום שני וכו
- parseDuration(tokens): משכים
- parseIntervals(tokens, now): מ..עד.. בין..ל..
- parseRecurrence(tokens, now): כל יום וכו
- parseAmbiguity(tokens, now): בערך לקראת מתישהו וכו
- parseTemporalHe(text, now, timezone): orchestrator שמחזיר אחד מהטייפים עם confidence
