NEW PRODUCT REQUIREMENT

Synco must distinguish between:
1. Questions that are critical right now for executing the user’s request.
2. Questions that are useful for learning the user better, but are not critical to execute the current request.

If information is critical to the current action:
- Ask now, in the current flow.
- Ask the smallest number of questions, ideally one focused question.

If information is missing but not critical:
- Do not block the user.
- Execute the safe action.
- Add a deferred open question to a dedicated app area.

Suggested area name:
- "מרכז ההיכרות"
Other possible names:
- "אזור ההשלמות"
- "מרכז הדיוק"
- "סינקו שואל"

Example:
User says:
"תזכיר לי לדבר עם דניאל על הפרויקט בשבת"

Synco should:
- Create/schedule the task for Saturday.
- Not block the user by asking who Daniel is or what project.
- Add deferred questions:
  - "מי זה דניאל עבורך?"
  - "על איזה פרויקט מדובר?"
  - "האם דניאל קשור לעבודה, לסינקו, למשפחה או למשהו אחר?"

When the user later answers in the dedicated area:
- Store the answer into memory/entity/profile.
- Close the answered question.
- Generate follow-up questions only if the new information creates a meaningful gap.

IMPORTANT INTELLIGENCE REQUIREMENT

Every new input to Synco should be checked against open questions.

Inputs include:
- typed free text
- voice input
- task creation
- task edits
- day planning commands
- task completion reports
- user answers in the dedicated questions area

If a new input contains enough information to answer an existing open question:
- Synco should link the answer to the question.
- Update memory/profile/entities.
- Mark the question as answered or resolved.
- Remove it from the active questions list.
- Create follow-up questions only when useful and not intrusive.

CORE PRODUCT RULE

Synco should not stop execution because information is missing for learning.
Synco should stop execution only when information is missing for doing the requested action correctly.

DESIGN GOALS

1. Reduce unnecessary real-time questions.
2. Avoid false conclusions.
3. Let Synco act quickly when safe.
4. Let Synco learn over time through deferred questions.
5. Convert every input into useful structured knowledge where possible.
6. Keep user trust by not pretending to know what it does not know.
