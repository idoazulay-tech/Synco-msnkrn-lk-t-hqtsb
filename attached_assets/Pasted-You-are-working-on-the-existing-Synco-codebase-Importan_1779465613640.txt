Important new Synco product rule:

A scheduled task block that elapsed without user confirmation is NOT proof that the task was completed and is NOT proof of actual duration.

Synco must distinguish:
- plannedDuration
- elapsedScheduledBlock
- actualDurationConfirmed
- actualDurationEstimated
- executionStatus: confirmed | estimated | unconfirmed_elapsed | not_completed

Product goal:
Many users will not press "completed" exactly when a task ends.
Synco must learn real duration with minimal friction, using later quick confirmation, voice update, timer evidence, or strong execution signals.
It must never learn actual duration merely because scheduled time passed.
