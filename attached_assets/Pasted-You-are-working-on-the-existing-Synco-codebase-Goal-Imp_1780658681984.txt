PART 1 — CREATE INTEGRITY GATE HELPER

Create:

server/brain/services/learningIntegrityGate.ts

Implement:

type LearningIntegrityStatus = 'accepted' | 'duplicate_ignored';

interface LearningIntegrityGateResult {
  status: LearningIntegrityStatus;
  priorEventId?: string;
  windowMs?: number;
  occurredAtDiffMs?: number;
  reason?: string;
}

Export:

runContextualDuplicateGate(event): Promise<LearningIntegrityGateResult>

Input can be typed locally if importing Prisma types is awkward.

Gate config:

task_completed:
- windowMs: 3000
- useDateIso: true

task_execution_completed:
- windowMs: 5000
- useDateIso: true

task_created:
- windowMs: 3000
- useDateIso: false

Logic:

1. If process.env.SYNCO_INTEGRITY_GATE_ENABLED === 'false':
   return { status: 'accepted', reason: 'disabled' }

2. If event.eventType is not in config:
   return { status: 'accepted', reason: 'event_not_gated' }

3. If event.taskId is missing:
   return { status: 'accepted', reason: 'missing_task_id' }

4. Build windowStart:
   event.occurredAt - windowMs

5. Query Postgres for a prior LearningEvent with:
   - same userId
   - same taskId
   - same eventType
   - occurredAt >= windowStart
   - occurredAt < event.occurredAt
   - id != event.id
   - if useDateIso is true and event.dateIso exists:
       same dateIso

6. For task_created:
   - do NOT require dateIso match.
   - same taskId + eventType within 3 seconds is duplicate.

7. If prior event exists:
   return:
   {
     status: 'duplicate_ignored',
     priorEventId: prior.id,
     windowMs,
     occurredAtDiffMs
   }

8. If none:
   return { status: 'accepted' }

9. On any gate error:
   console.warn('[LearningIntegrityGate] gate_error_fail_open', safe metadata only)
   return { status: 'accepted', reason: 'gate_error_fail_open' }

Important:
For recurring task completions, dateIso protects different occurrences from being treated as duplicates.
If useDateIso is true but event.dateIso is missing, still run the gate using userId + taskId + eventType + time window only, but set reason/metadata internally if useful. Do not reject the event just because dateIso is missing.
