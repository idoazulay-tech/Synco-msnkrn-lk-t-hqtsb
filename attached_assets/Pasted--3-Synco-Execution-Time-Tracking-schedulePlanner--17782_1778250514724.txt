חשוב:
אל תבנה UI חדש גדול.
אם כבר יש כפתור התחלה או טיימר, התחבר אליו.
אם אין, הוסף כפתור קטן רק במקום הכי טבעי, למשל TaskDetail או Home current task:
"התחל משימה"

כאשר המשתמש מתחיל משימה:
- שמור actualStartTime זמני ב-store או local state.
- שלח LearningEvent:
  eventType: task_started
- אם הלוגינג נכשל, הפעולה לא נשברת.

כאשר המשתמש מסמן משימה כהושלמה:
- אם יש actualStartTime, חשב actualDurationMinutes.
- אם אין actualStartTime, אפשר להשתמש ב:
  actualStartTime = startTime המתוכנן אם קיים
  או לא לחשב actualDurationMinutes ולשמור metadata חלקי.
- שלח LearningEvent:
  eventType: task_execution_completed
- בנוסף השאר את task_completed הקיים כמו שהוא.
- לא למחוק task_completed.
- task_execution_completed הוא שכבת עומק נוספת.

- אל תשבור את response הקיים.

Response daily-summary צריך לכלול את executionStats בלי לשבור שדות קיימים.

בדיקות edge cases:
- actualEndTime לפני actualStartTime
- plannedDurationMinutes חסר
- actualStartTime חסר
- taskId חסר
- משימה completed בלי start event
- כמה start events לאותה משימה
- task_execution_completed כפול
- metadata ריק

התנהגות edge cases:
- אם actualEndTime לפני actualStartTime, אל תשמור duration שלילי.
- החזר warning או שמור metadata בלי actualDurationMinutes.
- אם יש כמה task_started לאותה משימה, השתמש באחרון לפי occurredAt.
- אם task_execution_completed כפול, לא לעצור כרגע, רק לתעד. נפתור dedup בשלב עתידי.
