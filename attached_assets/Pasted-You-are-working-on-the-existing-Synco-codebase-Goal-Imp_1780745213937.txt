Persist only when:
- the request successfully created or planned something,
OR
- the question is not required to proceed immediately,
OR
- the question is clearly useful for learning context/entity/project/domain.

Do not persist when:
- response commandType is ask_clarification and the system is already asking the user inline because execution is blocked.
- needsClarification is true and no action was executed.
- the question is only a transient conflict-resolution question that must be answered now.

- 'time_preference' if question asks about preferred time
- 'domain_classification' if question asks whether it is work/family/Synco/etc.
- 'ambiguity_resolution' only if not blocking

priority:
- high for entity_identity or project_identity
- normal for task_context/domain/location
- low for general preference questions

blocking:
- false for all Phase 2a persisted questions

sourceInputText:
- truncate to <=100 chars through existing service behavior

sourceInputRoute:
- 'planning'

- or the actual route name

- short factual reason, for example:
