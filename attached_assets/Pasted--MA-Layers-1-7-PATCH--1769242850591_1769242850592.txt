אני ממשיך את פרויקט MA שנבנה לפי כל הפרומפטים הקודמים (Layers 1-7).
עכשיו אני רוצה לבצע PATCH מדויק שמוסיף רק את מה שחסר כדי שכל היכולות המעודכנות שסיכמנו יעבדו מקצה לקצה, בלי להמציא פיצ’רים חדשים ובלי לשנות את הארכיטקטורה הקיימת.

הדגש: להוסיף 6 השלמות קריטיות שחסרות כרגע:
1) lastQuestionContext ב ContextManager כדי שתשובות קצרות ("כן/לא/2/מחר") ייקשרו לשאלה הנכונה
2) pendingPlanProposal ב Store עבור Plan A/B (Reshuffle) + UI לאישור תכנית
3) taskType taxonomy + classifier בסיסי כדי ש Layer 5 (Learning) יוכל לבנות situationKey יציב ולא להתפוצץ
4) מדיניות זמן (Time Parsing Policy) ל MVP בלי ספריות חיצוניות, עם הגדרה ברורה מה נתמך ומה לא
5) Recurrence בסיסי לאימונים קבועים (weekly_fixed) כדי להתיישר עם ההחלטות שלנו על אירועים חוזרים
6) feedbackCooldowns כדי למנוע spam והצפה (No Spam policy בשכבה 7)

אסור:
- לא להוסיף שירותים חיצוניים חדשים
- לא להוסיף DB אמיתי
- לא להוסיף מנגנון דיבור/אודיו
- לא להוסיף יכולות שלא דיברנו עליהן

כל התיקונים חייבים להיות מחוברים ל Layers הקיימות, עם tests, ועם UI שמציג פרקטית.

========================
A) PATCH 1: lastQuestionContext (Layer 2 + 3 + Server)
========================

דרישה
להרחיב את /src/layers/intent/memory/ContextState.ts ואת ContextManager כך שישמר:
lastQuestionContext: {
  questionId: string,
  expectedAnswerType: "choice|free_text|time|date|duration|confirm|plan_choice",
  targetField: "date|time|duration|scope|targetCancel|clarifyIntent|plan_choice|unknown",
  relatedEntityType: "task|event|none",
  relatedEntityId: string | null,
  askedAtIso: string
} | null

חוקים
1) בכל פעם ש DecisionEngine מחזיר decision=ask, הוא חייב לקרוא ContextManager.setLastQuestionContext(...)
2) כש Input קצר מאוד ("כן", "לא", "מחר", "2", "20 דקות"), Intent Engine יסמן context.isFollowUp=true
3) Decision Engine חייב לדעת לפרש תשובת follow-up לפי lastQuestionContext:
- אם targetField=time: "2" -> time entity
- אם targetField=date: "מחר" -> date entity
- אם targetField=confirm: "כן/לא" -> boolean
- אם targetField=plan_choice: "A/B" או "תכנית 1/2" -> plan selection

מימוש נדרש
- להוסיף util חדש: /src/utils/answerParsers.ts
  parseAnswerByContext(answerText, lastQuestionContext) -> partialEntities + flags
- לעדכן Layer 2: אם detected follow-up, עדיין להחזיר IntentAnalysis רגיל, אבל להוסיף context.refersToPrevious=true
- לעדכן Layer 3: לפני decide, אם follow-up ויש lastQuestionContext, לבצע merge:
  mergedEntities = merge(lastEntities, currentEntities, parsedAnswerEntities)

Tests
- להוסיף טסטים ב intentEngine.test.ts עבור "כן", "לא", "מחר", "2", "20 דקות"
- להוסיף טסטים ב decisionEngine.test.ts שמדמים lastQuestionContext ומוודאים ש follow-up מוביל ל execute

========================
D) PATCH 4: Time Parsing Policy ל MVP (Layer 2 + 4)
========================

מטרה
בלי להכניס ספריות חיצוניות, נגדיר מה נתמך כדי שזה יעבוד יציב.

TimePolicy (חובה)
להוסיף /src/layers/intent/rules/timePolicy.ts שמגדיר:

נתמך:
1) שעות בפורמט מספרי:
   "ב-2" => 02:00 או 14:00 לפי הקשר (להלן)
   "14:00"
2) "היום", "מחר", "מחרתיים"
3) ימים בשבוע: "ראשון..שישי" (להחזיר רק raw אם לא ניתן לחשב dateIso)
4) durations:
   "20 דקות", "חצי שעה"=30, "שעה"=60, "שעתיים"=120
5) relative:
   "עוד 20 דקות" => durationMinutes=20 (לא timeIso)

חוק חשוב לשעה "2" בעברית
אם אין AM/PM:
- אם המשתמש כתב "בצהריים/אחהצ/ערב/בוקר" להשתמש בזה
- אחרת: אם עכשיו (server time) אחרי 12:00, לפרש "2" כ-14:00, אם לפני 12:00 לפרש כ-02:00? לא.
כדי למנוע טעויות, במצב כזה:
- להחזיר time.raw="2"
- time.iso=""
- confidence נמוך
- missingInfo יכלול "time_format_clarification"
ואז DecisionEngine ישאל: "2 בבוקר או 2 בצהריים?" עם options.
(זו עדיין שאלה אחת.)

חיבור לשכבה 4
Scheduler דורש iso כדי לשבץ.
אם אין iso, TaskTimeEngine לא משבץ, אלא מחזיר ask דרך Decision flow.

Tests
- tests ל parsing durations
- tests ל "ב-2" עם "בערב" => 14:00? לא. בערב => 20:00 אם כתוב 8. לכן כאן רק מגדירים: "ב-2 בערב" לא תקני. במצב כזה low confidence + ask.
- tests ל "14:00" => iso תקין
- tests ל "עוד 20 דקות" => durationMinutes=20
